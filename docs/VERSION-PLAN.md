## 插队机制

- **patch 版(v0.x.y)**:走查修复、owner 即时反馈、竞品小动作——随时做,当天上,不等 minor 排期
- **minor 版(v0.x)**:按上表主题推进;owner 可在任意版本间调整优先级,说一声就重排
- **持续开发**:版本之间不中断——一个版本冒烟/部署/提交后,立即进入下一个;owner 的新需求随时插入当前或下一版本
- **设计锦标赛产物**(R1-A/B/C)按版本落入:v0.5/0.6 吃 R1-A,v0.7 吃 R1-B,v0.8 吃 R1-C;锦标赛发现与版本实现冲突时,以落地实测为准并回写锦标赛文档
- 每周一竞品 cron 的「借鉴池」进入最近一个 minor 版评估

## 评审反击机制(owner 2026-07-20 钦定)

评审轮次不止于「R1 设计 → R2 红蓝对抗 → R3 终裁」。新增铁律:

1. **终裁若含「大退让」**(砍掉/推迟 owner 明确要的能力、放弃关键指标、接受不可接受的妥协)→ **必须触发反击型评审(R3.5)**:针对退让点专项寻找更优解,不许直接认栽。
2. 反击轮仍未找到更优解 → 记录「为什么不做的理由」进 decisions.md 才可退让;找到 → 修订终裁并注明反击轮出处。
3. 可循环:反击轮的方案再过对抗评审,直到收敛。
4. 首次适用:R2 红蓝评审建议「chatgpt adapter 整条线推迟」→ 与 owner「接入 ChatGPT」原话冲突 → 已触发 R3.5(2026-07-20,见 design-tournament/R3.5-counterattack-chatgpt.md)。
