# MULTI-HARNESS-DESIGN — 多设备 × 多 harness 统一接入与监控沟通体系(终稿)

> 设计锦标赛 Round 3 综合终稿(主代理终裁)· 2026-07-20
> 输入:R1-A(架构)/ R1-B(监控大盘)/ R1-C(沟通协议)+ R2 红队(5 致命/15 重要)与蓝队(6 致命/12 重要)评审。
> 裁决方式:红蓝共识项直接焊死;分歧项给出裁定与理由;与 VERSION-PLAN 对齐落地版本。
> 三条地基缝(红队收口语):**等级语义两义、状态存储无家、通知管道无归属**——本终稿首先补齐这三条缝。

---

## 0. 终裁摘要(9 条)

1. **统一分级真源**:一张正交矩阵(事件 × 屏显级 × 触达级),B 的阻塞语义为准,C 降级为通道消费者。物理形态 = 一份机器可执行配置文件(`grading.config.json`),设备侧唯一持有。
2. **阻塞型即时性(推翻 R1-C 批送)**:审批/提问(阻塞型)即时单发,靠冷却+合并防轰炸;信息型(完成/自愈/额度)才走 30min 批送。agent 空转 60 分钟换通知聚合是亏本买卖。
3. **红线防误批 + 可撤销**:红线项强制二次确认(点一次变确认态);普通项拍下后 10s 可撤(乐观延迟执行);已执行回执带指引深链。
4. **notifier 组件归位**:定级引擎放在**设备侧**(与 adapter 同机),订阅本机事件流,持唯一定级配置,出站只发脱敏负载。网关保持零解析零存储正文(D-11 不破)。拍板 collector 复用 interactive-decision-h5 资产,飞书 app secret 记入凭证分层表。
5. **状态账本落点**:网关侧加一张轻量元数据表「状态迁移 + 已读游标 + 上次打开时间」(只存元数据不存正文,与"不建中心事件库"不冲突——修订 R1-A §3.2 的否决范围并记 D-12)。
6. **游标语义修正**:jsonl adapter 对外 seq = adapter 内部分配的单调序号(内存+checkpoint 持久化),内部按文件维护 {inode,offset} 集合,二者解耦;codex fork 族诚实暴露为独立会话;gap 直接发 `resync_required`,不发明新协议。
7. **adapter 认证前置**:machine token(或最低限度 adapter bearer)从 P3 提前到 **P1**——adapter server 取代 kimi server 成为隧道唯一上游之日,就是认证生效之日,不接受零认证窗口期。
8. **状态词表统一**:机器态随便几态,用户可见词表收敛 6 个:**等你批准 / 等你回答 / 失败了 / 卡住了 / 完成了(未验收)/ 未知**;三份文档共用。stalled 分两档:30min 灰底观察(P2),2h 升 P0;`awaiting_input` 永不判 stalled;codex「等本机键盘」单列文案,不给错误动作。
9. **里程碑现实化**:P1 砍为「adapter server + kimi/claude/codex 三读路径 + harness 命名空间(单机)+ adapter 认证 + 修正后 tailer 单测」,4-6 周;zcode/workbuddy 只读探针 best-effort 不验收;机器维度留 P3。

## 1. 统一分级矩阵(唯一真源 · grading.config.json 的逻辑表)

纵轴 = 事件(跨 harness 统一类型);横轴两个**正交**维度:屏显级(大盘怎么排)× 触达级(通知怎么走)。一事件一格,三份文档(R1-B/R1-C/实现)引用同一张表,删掉各自为政的映射。

| 事件 | 屏显级(大盘) | 触达级(通知) |
|---|---|---|
| 审批请求·红线(钱>¥1000/生产写入/凭据/对外不可逆) | P0 置顶(阻塞时长序) | **即时单卡 + 二次确认**;夜间:只推一次,首次活跃置顶重推 |
| 审批请求·普通(阻塞会话) | P0 | **即时单发**(冷却 10min/会话,同类合并);夜间攒早报 |
| 提问·阻塞(带建议选项) | P0 | 即时单发(同上) |
| 提问·无建议开放问 | 不上屏(退回 agent 补建议) | 不许发 |
| 任务失败·终态非零退出 ∧ 无自动重试 | P0 | 即时单发(合并同类) |
| 僵死 stalled(非 awaiting_input ∧ 最后事件>2h) | P0 | 即时单发 |
| 疑似僵死(>30min 无事件)| P2 灰底观察 | 不触达 |
| 等本机键盘(codex TUI 活跃互斥)| P0(文案:这台 Mac 上有东西等你按) | 即时单发 |
| 完成·需验收(任务发起时类型决定,不信 agent 自报) | P1 待过目 | 批送(30min);对客 deadline<次日 10:00 的 7:30 单发 |
| 完成·常规 | P1 | 批送 |
| 异常已自愈(重试成功/隧道自愈) | P1(一行) | 不触达 |
| 额度 ≥80% | P1 | 批送;≥95% 影响在途 → 即时 |
| 机器离线·有在途任务 | P0 异常区 | 即时;卡住的是红线审批 → 同红线 |
| 机器离线·无在途 | P2 异常区 | 不触达 |
| 排队中(自己发的 prompt 入队) | P2(动作回执) | 不触达;超 30min 升观察 |
| 运行中/空闲/心跳/cron 成功 | 不上屏(聚合摘要行) | P3 永不 |

防轰炸硬机制(保留 R1-C 全部):单会话冷却、同类合并、批送节律、夜间静默(破静默清单显式枚举:红线/额度95/卡住P0的离线)、**系统级熔断**(任意通道每小时 >N 条 → 自动降级聚合摘要+告警)、夜间 P0 只推一次。
指标修订(蓝队 Y5):撤掉「打开率≥90%」(Goodhart),改双约束:误报率(标"不该发")<10% + 漏报(每周定性回顾);「未打开但 24h 内无需处理」记正向。

## 2. 修订后拓扑(补三条缝)

```
手机 H5 ──HTTPS/WSS(JWT)──▶ 阿里云网关(your-gateway)
  ├─ allowlist 2.0:路径模板 × 能力位(/m/:machine/h/:harness/)
  ├─ 元数据表(新):用户 / machine token / 状态迁移+已读游标+上次打开(D-12)
  └─ 注册表:machine→隧道端口、能力清单、最后心跳
        ▲ 每机一条 SSH -R(tunnel.sh 自愈机制沿用)
        │ 应用层认证:machine token(P1 起强制)
  ┌─────┴────────────────────────────────────┐
  │ 每台设备(M5/M1/M2/ECS)                   │
  │  agent = adapter server(loopback,认证)   │
  │    ├ adapters: kimi(L3) claude(L3)      │
  │    │   codex(L1+exec臂) zcode/workbuddy  │
  │    │   (L1 探针) chatgpt(快照,缓议)     │
  │    ├ state store(游标/会话级状态摘要)    │
  │    └ notifier(唯一定级配置+脱敏出站)     │
  │         → 飞书 collector(拍板闭环)       │
  └──────────────────────────────────────────┘
```

- **心跳升级**:控制面心跳携带「会话级状态摘要(6 词态+计数+脱敏标题)」——大盘首屏聚合的必要条件(红队矛盾项,修订 R1-A §3.3)。
- **notifier 在设备侧**:订阅本机 adapter 事件流 → 过 grading.config → 脱敏 → 出站。定级配置只有一份,每台机镜像持有;通知负载只含「机器·项目·事件类别·脱敏标题+深链」,审批参数/问题正文永远只在 H5(JWT 后)展示;锁屏文案统一无内容版。
- **拍板闭环**:飞书卡片按钮(普通项 1 击+10s 可撤)→ collector 落账 → 回写会话注入;红线 → 二次确认卡;机器离线时决策**在网关排队**,回执写明「已记账,待机器上线执行」(蓝队 Y7)。

## 3. 适配层修订要点(并入 R1-A)

- **SessionSummary 增补**:`hasOpenTask`(stalled 判据)、`terminal/receipt` 事件类型(done_unread 判据)、`statusConfidence`(保留,heuristic 必须诚实暴露)。B 的 9 态 ↔ A 的状态模型做逐态映射表;只读 harness 收缩为可判子集并显式标注。
- **claude adapter**:待审批/进行中会话**豁免 30min 回收与 LRU 淘汰**(蓝队 Z4);resolveApproval 冷恢复路径(进程不在 → resume + 重放 pending approval)进 P2 验证。
- **codex adapter**:TUI 活跃判定改「进程存活检测 + mtime」双条件,宁可错锁不可错放;exec 臂补延迟预算(P95 首 token),超预算降级为「只读+深链回本机」。
- **chatgpt**:三层快照源组合(E `codex cloud` 官方 CLI 已实测可用 + B Playwright 快照 spike 门控可弃层 + D/F ingest 兜底),进 v0.5 骨架+E+D+B-spike / v0.6 B(spike 通过才做,kill-switch 预写)——R3.5 反击轮(2026-07-20,docs/design-tournament/R3.5-counterattack-chatgpt.md)推翻了原「整条线推迟到 P3 后」的退让;红蓝判对的是「人工导入该死」,判错的是把它当唯一形态。B 层 ToS 灰色风险需 owner 签收后动工。
- **zcode/workbuddy**:只读 probe,产出 probe 报告 + degraded_reason 正确暴露即为验收;升级逐案后议。
- **脱敏管线**:分两层写明边界——凭证形态 best-effort(fixture 召回 ≥95% 进 P2 验证);业务敏感靠访问控制兜底(snapshotOnly 默认 member 不可见+reveal 审计)。

## 4. 大盘修订要点(并入 R1-B)

- **Now 视图做硬**(90% 价值所在);**Today/Week/Away 顺延**到状态账本落地后(红队 70% 裁决),Today 加「今夜在跑」前瞻节(蓝队 X3)与「今日额度水位」(X7)。
- P0 区视觉合并为一组「等你处理」(待审批/待回答不分色);行内按 capability 渲染:只读 harness →「请回本地处理」不给错误按钮。
- S7 验证改 12-15 会话可达规模;S1 只测「找出+排序正确」,处理另测 90s 且强制抽一条「看上下文后再批」。
- 隐式已读:滚动停留 ≥5s 且覆盖结果区才清;输入框聚焦不算阅读;留「标回未读」。

## 6. 未解与延后(诚实清单)

1. claude Agent SDK `listSessions/getSessionMessages` 是否公开稳定 API → P1 探针确认,不稳定则直接读 jsonl。
2. 机器休眠与故障的区分:心跳最后上报 lid/battery,元数据标 sleep/unreachable(蓝队 Y7 部分采纳);边缘加密快照(opt-in)另开决策。
3. 飞书已读回执采集路径未验证 → 「深链点击率」(H5 打点)替代打开率。
4. ECS 装 agent 与 zaios 侧 notify 脚本收敛的负责人/工作量,进 v0.8 范围时单列。
5. 多会话抢答归并(工具名+参数指纹+项目为归并键,一次拍板广播同组)→ v0.8 拍板协议细化。
