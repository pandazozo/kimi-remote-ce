## 3. 王佐真正会每天用的是什么

诚实回答：**每天真用的只有三样** —— ① 飞书 P0 单卡（红线+失败），命根子，前提是别吵；② 大盘 Now 视图的顶栏 + P0 区，5 秒扫一眼就走，安心成本确实低，这是 B 最大的功劳，必须保住；③ 非红线审批的行内批准。**会用两周然后死掉的**:Today/Week 视图（新鲜感过了就不看）、Away 摘要（顶栏已经回答了）、chatgpt 快照导入（永远不会导）、72h 自动已读（被我发现一次「东西自己消失了」，整个未读体系的信任就崩了）。**会被我亲手改掉的**:P1 30min 批送（第一次被 agent 空转气到就会改成即时）、打开率考核。这个系统的死亡方式只有一种：不是漏报，是吵 —— 一次夜间四连震、或连续三天假 stalled，我就会把通知全关，协议归零。所以下一轮的优先级不是补 Today/Week，而是焊死三件事：**统一分级真源（Z1)、阻塞型事件即时性（Z2)、防误批+可撤销（Z3)**。其余都是有了这三样之后的增量。
